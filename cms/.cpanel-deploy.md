# Santa Maria CMS — DomaiNesia cPanel Deploy Guide

Target: subdomain `internal.santamaria.co.id` on shared cPanel hosting (account `santama2`).

## 1. Create subdomain in cPanel

- cPanel → Domains → Create a new domain.
- Domain: `internal.santamaria.co.id`.
- Do NOT let cPanel put the docroot inside `public_html`. Use a custom path:
  `/home/santama2/internal.santamaria.co.id/` (cPanel default) — we will override it later.

The final docroot we want is `/home/santama2/cms/public`. Two options:

**Option A (recommended): symlink after upload.**
Let cPanel create its default folder, then in Terminal replace it with a symlink:
```
rm -rf /home/santama2/internal.santamaria.co.id
ln -s /home/santama2/cms/public /home/santama2/internal.santamaria.co.id
```

**Option B: edit domain docroot in cPanel → Domains → Manage → set to `/home/santama2/cms/public`.**

## 2. Create MySQL database

- cPanel → MySQL Databases.
- Create database: `santama2_cms`.
- Create user: `santama2_cms` with a strong password.
- Add user to database with ALL PRIVILEGES.
- Note down: DB name, username, password.

## 3. Upload the Laravel app

Put the app at `/home/santama2/cms/` (outside `public_html`).

**Method 1: ZIP upload via File Manager.**
1. Locally: `cd "/Users/juchr/Documents/Santa Maria"` then
   `cd cms && zip -r ../cms.zip . -x "node_modules/*" ".git/*" "storage/framework/cache/*" "storage/framework/sessions/*" "storage/framework/views/*" "storage/logs/*"`
2. In cPanel File Manager, upload `cms.zip` to `/home/santama2/`.
3. Extract; resulting folder: `/home/santama2/cms/`.

**Method 2: SSH + git + composer (if SSH is enabled).**
```
ssh santama2@internal.santamaria.co.id
cd ~
git clone <your-repo-url> cms
cd cms
```

## 4. Install PHP dependencies

In cPanel Terminal (or SSH):
```
cd /home/santama2/cms
composer install --no-dev --optimize-autoloader
```

If `composer` isn't on PATH, use `/usr/local/bin/composer` or activate PHP Selector:
```
/opt/cpanel/ea-php82/root/usr/bin/php /usr/local/bin/composer install --no-dev --optimize-autoloader
```

## 5. Configure `.env`

```
cd /home/santama2/cms
cp .env.example .env
nano .env
```

Fill in DB credentials. Confirm:
```
APP_URL=https://internal.santamaria.co.id
APP_ENV=production
APP_DEBUG=false
DB_DATABASE=santama2_cms
DB_USERNAME=santama2_cms
DB_PASSWORD=<password>
SESSION_DOMAIN=.santamaria.co.id
```

Then:
```
php artisan key:generate
```

## 10. Point subdomain docroot

As described in step 1 — symlink or cPanel docroot override so `https://internal.santamaria.co.id` serves `/home/santama2/cms/public`.

## 11. Verify

- Visit `https://internal.santamaria.co.id/login` → login form.
- Log in as admin, create a test article with `status=published`.
- From the landing page origin: `curl https://internal.santamaria.co.id/api/v1/public/articles` → expect `{"success":true,"data":{...}}`.

## 12. Landing page wiring

Landing page at `https://santamaria.co.id` fetches from `${window.location.origin}/api/v1/public/...`.
Since origin is `santamaria.co.id` (not `internal.santamaria.co.id`), you must either:
- (a) Add an Apache rewrite on the main domain: `/api/v1/public/*` → proxy to `https://internal.santamaria.co.id/api/v1/public/*`, OR
- (b) Change the landing page to call `https://internal.santamaria.co.id/api/v1/public/...` directly (CORS is already allowed).

## Troubleshooting

- **500 error**: check `storage/logs/laravel.log`. Usually permissions or `.env` missing `APP_KEY`.
- **Session/login fails across subdomain**: ensure `SESSION_DOMAIN=.santamaria.co.id` and HTTPS is working.
- **Images 404**: re-run `php artisan storage:link`.
- **CORS blocked**: confirm `config/cors.php` lists the exact origin, then `php artisan config:cache`.
